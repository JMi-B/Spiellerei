# Java

## Objekte
Java ist eine Objektorientierte Strache.
Ein Objekt ist eine Instanz einer Klasse.
	Die Klasse ist die Schablone, das Objekt ist die Ausführung
	Um ein Objekt zu erstellen:
	`Klasse objektName = new Klasse();`
## Methoden
- Codeblock, der nur ausgeführt wird, wenn er aufgerufen wird
- Methoden haben Paramenter
- Methoden sind #funktion 
- Methoden können mehrfach verwendet werden.
- Erstellung:
	- eine Methode muss in einer Klasse deklariert werden
	- `methodName()`
	- `static` - die Methode gehört zur übergeordneten Klasse
	- `voit` - die Methode hat keinen Rückgabe wert.
- Aufruf:
	- `methodName();

- Zahlen werden nicht in Anführungszeichen gesetzt.
- Innerhalb der Methode können auch Rechungen durch geführt werden.

# Vererbung
## Aufgaben Beschreibung
Als Objekt orientierte Programmiersprache arbeitet Java mit Klassen Objekten und Vererbung. Aufgabe ist es ein System von Klassen und Unterklassen zu erstellen in dem Eigenschaften vererbt werden. Eine Klasse definiert allgemeine Methoden für alle. Die Unterklassen spezifizieren die Eigenschaften. Ziel ist ein array aus dem spezifisch ausgelesen wird.
### Beispiel von Georg
`public static void main(String[] args)`
    `{`
        `Schueler schuelerA = new Schueler();`
        `schuelerA.Alter = 14;`
        `schuelerA.Schulklasse = "Klasse 9a";`
        
        `Person personA = new Person();`
        `personA.Alter = 20;`
        
        `Lehrer lehrerA = new Lehrer();`
        `lehrerA.Alter = 35;`
        `lehrerA.Fach = "Englisch";`
        
        `Person[] personen = new Person[3];`
        `personen[0] = schuelerA;`
        `personen[1] = personA;`
        `personen[2] = lehrerA;`
        
        `for (Person person : personen){`
            `System.out.println(person.Alter);`
            `if (person instanceof Schueler){`
                `String schulklasse = ((Schueler)person).Schulklasse;`
                `System.out.println(schulklasse);`
            `}`
        `}`
    `}`
## Akzeptanzkriterien
#vererbung #subclass #superclass
- Eine klasse mit einem allgemeinen Kriterium
- zwei subclasses mit mit jeweils einem Unterschiedlichen Kriterium, das von der Oberklasse abhängt
- eine Klasse die die subklasse ein der Unterklassen ist
- Eine List Objekten für jede Klasse eins.
- Ausgabe eines der objekte aus der oberklasse herraus
- Die Klassen müssen logisch von einander abhängen
