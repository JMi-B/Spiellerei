# Views und Stored Procedures
#sql #dbaufbau
Eine Datenbank besteht aus verschiedenen Elementen mit denen die Daten gespeichert, organisiert, verwaltet und abgefragt werden. In einer `sql`-basierten Datenbank gibt es die drei Tabellen, Views und Stored Procedures.
## Tabellen
Tabellen enthalten und ordnen die Daten. Sie sind das Gerüst. Sie bestehen aus Spalten und können sehr umfangreich sein (z. B. die `Users` Tabelle in der Swyx Ware). Sie sind untereinander verknüpft. Sie sind draufausgelegt von Computerprogrammen gelesen zu werden. Wenn die Daten normalisiert sind Zusammenhänge von Inhalten nicht offensichtlich. Die `InternelNumberd` Tabelle der Swyx Ware enthält keinen Namen. Dieser steht in der `Users` Tabelle. In meinem Datenbank Projekt sind die Medienarten von einender getrennt. Die Tabelle Medium vergibt einen eindeutige Nummer und bestimmt die Medienart. Sie enthält keine Information über Buch, Zeitschrift oder Datei.
Das durchsuchen von mehren Tabellen für Informationen, die oft zusammen gebraucht werden kostet Ressourcen. 
## Views
#Views 
View sind Teil der Datenbank und werden in ihr gespeichert. Views können ähnlich wie Tabellen genutzt werden. Sie sind vorprogrammierte Abfragen. Die in der Datenbank selber gespeichert werden. Sie stellen Daten zur Verfügung jedoch verändern diese nicht. Daten aus verschiedenen Tabellen können mit Views zusammen geführt werden. Bestimmte Daten aus umfangreiche Tabellen können übersichtlicher dargestellt werde. Informationen, die oft zusammen gesucht werden, können so schneller abgerufen werden. Daten werden im Arbeitsspeicher gehalten, weil die Datenbank weiß, dass sie zusammen genutzt werden,  obwohl sie für Datanbankmanagmentsystem getrennt sind. 
View sind also vordefinierte Abfragen. Diese werden bei jeder Benutzung mit den aktuellen Daten gefüllt. Dadurch muss der Benutzer weniger über das ganze komplexe Datenbanksystem wissen, wenn er seine Abfragen formuliert. Das kann auch ein Nachteil sein, der Aufwand einer Anfrage dadurch unterschätzt wird. Auch kann es zu Ungenauigkeiten kommen, wenn z.B. auf einer `View` Daten geändert oder gelöscht werden sollen.
## Stored Procedures
#Stored_Procedures 
StoredProcedures sind in der Datenbank gespeicherte `sql` Codes, die immer wieder verwendet werden können. Häufig benutzte Queries müssen nicht immer neu geschrieben werden. Es können zur Ausführung auch Parameter an sie übergeben werden. Auch komplexe Queries können als Stored Procedures gespeichert werden, z.B. Löschen und Anlegen von Usern um sicherzustellten das in allen Tabellen die Daten verändert werden.  Auch die Abfrage Geschwindigkeit kann durch Stored Procedures verbessert werden, da weniger Austausch zwischen Datenbankclient und DMS statt findet. Sie können auch zur Sicherheit beitragen, wenn Datenveränderung und Löschung nur über Stored Procedures ermöglicht wird
## Fazit
Um eine Datenbank kennen zu lernen und mit ihr zu arbeiten, neben der Struktur auch die Views und Stored Procedures betrachten. Sie zeigen, wie mit der Datenbank geabereitet wird.
